# 翻译清单 · 术语表与译文约定

本文件记录译文的**质量约定**：术语怎么统一、哪些保留英文、踩过哪些真错译。
译文本体在 `资源脚本/` 下的各 `map*.json` 和 `斜杠命令-中文说明.json` 里，是可二次编辑的资产。

---

## 一、术语统一表（最重要）

同一个英文概念，全程必须用同一个中文译法，不能时而这样时而那样：

| 英文 | 统一译法 | 说明 / 易错点 |
|---|---|---|
| response | **回复** | 不译"回答/响应"。`Good/Bad response` = 回复质量好/差 |
| effort（level） | **思考力度** | 指模型推理投入，**不是**用户的"投入程度" |
| agent / subagent | **智能体 / 子智能体** | 不混用"代理/agent"，大小写不混排 |
| session | **会话** | `New/Delete/Rename session` = 新建/删除/重命名会话 |
| credits | **额度 / 积分** | 按上下文取一致 |
| plan mode | **计划模式** | 与 Auto/Ask 模式成组 |
| output style | **输出风格** | 不译"输出样式"（同概念须一致） |
| land / landing | **代码合入 / 合入队列** | ⚠️ **真错译警示**：不是"上线/发布"！land = 把 PR 合进主干 |
| hooks | **钩子**（或保留 `Hooks`） | 技术术语，正文可保留英文 |

## 五、按约定保留英文（不汉化）

- **技术术语**：`Hooks`、`MCP`、`CLAUDE.md` 等约定俗成的。
- **Monaco 编辑器内部命令**：`Find and Replace`、`Fold Unchanged`、`Source Action...` 等（普通用户看不到，藏在编辑器命令面板）。
- **JSON Schema 描述**：扩展配置项的 schema 文本（开发者向）。
- **斜杠命令名本身**：`/login`、`/issue` 等（命令名不译，只译描述）。
- **`(internal)` / `[internal]` 内部项**：研发用，保留。
- **代码标识符**：任何会被当变量/属性 key 用的串。

---

## 六、译文交叉审（推荐）

译完后建议用第二个 AI（如 Codex/GPT、另一个模型）做独立复审，专挑：

- **真错译**（如 land/landing 被误成"上线"、`without stopping` 被漏译语义）
- **术语不统一**（同概念多种译法）
- **中英标点混用**（中文应全角）

本工具包的译文已经过一轮交叉审，上面术语表里的"易错点"多来自那次复审的发现。

---

## 七、二次编辑指引

- 改 UI 译文：编辑 `map1`/`map2`/`map3`/`map4`，重跑 `一键汉化.py`。
- 改命令描述：编辑 `斜杠命令-中文说明.json`，重跑（注入器会从干净基线重生成）。
- 补第三方技能命令：参照 `斜杠命令-第三方技能示例.json` 的写法，把 `"命令名":"中文描述"` 并入 `斜杠命令-中文说明.json`。
